# NaatuMithra: Multi-City Product Definition

NaatuMithra is a contextual companion designed to help users navigate the nuanced landscape of 30 Indian cities. It focuses on:
- **Linguistic Nuance**: Translating local slang into understandable English/emojis.
- **Gastronomic Moods**: Mapping how you feel to what you should eat.
- **Micro-Mobility**: Predicting the best way to travel based on hyper-local data.
- **Cultural Heritage**: Providing bite-sized heritage facts that are shareable.
- **Flavor Mapping**: Connecting local delicacies to global mental models.
- **Sonic Identity**: Immersive audio experiences capturing city vibes.

  - Context: "Commonly used among friends, similar to 'dude' or 'bro' in English"

### 2. Food Mood Mapper
**Purpose**: Suggest local dishes based on emotional state and city

**Acceptance Criteria**:
- Input: Mood (adventurous, comfy, hurry, royal) + City
- Output: Local dish + description + global equivalent + cultural context
- Bilingual descriptions
- Flavor notes and ingredients

**Moods Supported**:
- **Adventurous**: Spicy, bold flavors, experimental dishes
- **Comfy**: Comfort food, familiar tastes, nostalgic meals
- **Hurry**: Quick, street food options, grab-and-go
- **Royal**: Premium, special occasion dishes, elaborate preparations

**Example**:
- Input: "Comfy" mood in Chennai
- Output: "Idli Sambar - soft steamed rice cakes with lentil stew. Global equivalent: Savory rice pancakes. Comfort food that feels like home."

### 3. Heritage Cards
**Purpose**: Tell stories of cultural landmarks with AI-generated narratives

**Acceptance Criteria**:
- Input: City + Heritage site (or auto-generate for city)
- Output: Historical fact + AI-generated "Neural Legend" + bilingual description
- Shareable format (image + text)
- Downloadable as image

**Data Required per City**:
- Minimum 2-3 major heritage sites
- Historical facts with dates
- Cultural significance
- Local legends/stories

**Example**:
- Input: Bengaluru - Vidhana Soudha
- Output:
  - Fact: "Largest legislative building in India, built in Neo-Dravidian style"
  - Neural Legend: AI-generated story connecting architecture to local culture
  - Bilingual: English + Kannada

### 4. Travel Chaos Oracle
**Purpose**: Predict best travel routes and modes based on mission type

**Acceptance Criteria**:
- Input: Origin + Destination + Mission (Aesthetic/Fast/Minimal Chaos/Healthy)
- Output: Best mode + time estimate + "Road Karma" score + environmental impact
- Bilingual recommendations
- Real-time considerations

**Missions Supported**:
- **Aesthetic Route**: Scenic, cultural landmarks, photo opportunities
- **Fastest Path**: Minimum time, direct routes
- **Minimal Chaos**: Least crowded, peaceful journey
- **Healthy Air**: Best air quality, green routes

**Road Karma Score**:
- Health: Impact on physical well-being
- Carbon: Environmental footprint
- Time: Efficiency rating
- Chaos: Stress level indicator

### 5. Flavor Mapper
**Purpose**: Connect local dishes to global equivalents with mental models

**Acceptance Criteria**:
- Input: Local dish + City
- Output: Global equivalent + flavor profile + ingredients + preparation notes
- Mental model visualization (local â†’ global mapping)
- Bilingual descriptions

**Example**:
- Input: "Dosa" (Chennai)
- Output:
  - Global Equivalent: "Savory rice crepe, similar to French crepe but fermented"
  - Flavor Profile: "Crispy, tangy, umami-rich"
  - Mental Model: "Think of it as a healthier, fermented version of a pancake"

### 6. Soundboard
**Purpose**: Immersive audio experience of city vibes

**Acceptance Criteria**:
- 3-5 audio clips per city minimum
- Categories: Morning, Evening, Street, Nature, Cultural
- Duration: 30-60 seconds per clip
- Shareable and downloadable
- Bilingual descriptions

**Categories**:
- **Morning**: Temple bells, street vendors, traffic sounds
- **Evening**: Market buzz, cultural performances
- **Street**: Auto-rickshaws, local chatter, food stalls
- **Nature**: Birds, water bodies, parks
- **Cultural**: Festivals, music, traditional sounds

- **Key Features Used**: Food Mood Mapper (95%), Flavor Mapper (90%), Slang Translator (70%)

## Data Schema

### City Profile Schema
```yaml
city:
  id: string (2-3 chars, lowercase, required)
  name: string (required)
  nickname: string (required)
  region: string (North/South/East/West/Central, optional)
  slang:
    - phrase: string (required)
      meaning: string (required)
      emoji: string (optional)
      context: string (optional)
  foods:
    - name: string (required)
      description: string (required)
      category: string (Street/Traditional/Modern, optional)
      global_equivalent: string (optional)
  transit:
    - mode: string (Metro/Bus/Auto/Train/Ferry, required)
      name: string (required)
      characteristic: string (optional)
  heritage:
    - id: string (required)
      title: string (required)
      fact: string (required)
      year_built: number (optional)
      significance: string (optional)
  soundboard:
    - title: string (required)
      category: string (Morning/Evening/Street/Nature/Cultural, required)
      duration: string (required)
      description: string (required)
  vibe: string (one-line city character, required)
```

### Validation Rules
- Each city MUST have minimum 3 slang phrases
- Each city MUST have minimum 3 foods
- Each city MUST have minimum 2 transit modes
- Each city MUST have minimum 2 heritage sites
- Each city MUST have minimum 3 soundboard clips
- Each city MUST have a vibe description
- All required fields must be non-empty
- IDs must be unique across all cities

## Quality Standards

### Content Accuracy
- All heritage facts must be verified from reliable sources (Wikipedia, historical records, government websites)
- Slang must be authentic and currently used (verified with native speakers when possible)
- Food descriptions must be accurate to regional cuisine (cross-referenced with cookbooks, local restaurants)
- Transit information must be current (verified with official transport websites)

### Cultural Sensitivity
- Respect regional identities and pride
- Avoid stereotypes and generalizations
- Acknowledge regional diversity within cities
- Include minority perspectives when relevant
- Use inclusive language
- Celebrate local culture without appropriation

### Bilingual Quality
- Native script must be accurate (verified with native speakers)
- Translations must preserve meaning and cultural context
- Phonetic transliteration must follow standard conventions
- Cultural nuances must be maintained in translation
- Emoji usage must be culturally appropriate

### Consistency
- Tone must be consistent across all cities (friendly, informative, "Naatu" authentic)
- Format must be standardized (same structure for all city entries)
- Terminology must be consistent (use same terms for similar concepts)
- Examples must be relatable and diverse
